# ZOM-BOY+ 设计 — 僵尸互斥 + 感染生成异地化 + AI 不胆小（破贴边墙)

日期：2026-05-16
状态：已与用户对齐，待 spec 复核 → 转 writing-plans
范围：规则改动（动核心玩法）+ AI 评估 retune + harness 验收门槛。一个内聚 spec，**必须打包实现**。

---

## 1. 问题（为什么要改）

### 1.1 现象
玩家把 8 只僵尸贴满棋盘一条边列（如第 0 列 row0..7）。该阵型**完全不可被跳杀**，导致幸存者**永远无法取胜**。

### 1.2 严格证明（这是规则缺陷，不是代码 bug）
跳杀几何：幸存者 S、相邻僵尸 Z、落点 `L = 2Z − S`，要求 L 在界内、非石头、空。
墙在第 0 列，`Z=(r,0)`。幸存者只能站第 0 或第 1 列（第 −1 列出界）。穷举：

| 幸存者位置 | 方向 | 落点 L | 结果 |
|---|---|---|---|
| `(r,1)` | 正西 | `(r,−1)` | 出界 ✗ |
| `(r−1,1)` | 西南 | `(r+1,−1)` | 出界 ✗ |
| `(r+1,1)` | 西北 | `(r−1,−1)` | 出界 ✗ |
| `(r−1,0)` | 向南 | `(r+1,0)` | 被下一只僵尸占用 ✗ |
| `(r+1,0)` | 向北 | `(r−1,0)` | 被上一只僵尸占用 ✗ |

竖跳要求幸存者站在墙列空格上；满列无空格 → 站不上去。**连续 ≥2 只贴边僵尸（含两端）全部不可跳杀；满列 100% 免疫。**

### 1.3 定性
静止墙不感染任何人 → 僵尸到不了 4 杀 → 撞 `STALE_PLY_CAP` 判平。结果是**双方都赢不了的强制平局/死锁**。对游戏而言"幸存者无论如何赢不了"即设计缺陷。修复属**规则层**，会动平衡，须 harness 复测。

## 2. 目标 / 非目标

**目标**
- G-A：消除"不可摧贴边墙"——任何僵尸结构都存在可被跳杀的着手点。
- G-B：保留感染玩法（夹击仍可达成）。
- G-C：僵尸 AI 不再在距离 2 干耗，敢合拢/敢拼交换（解决原"AI 太胆小"）。
- G-D：harness 量化，整体平衡仍健康（见 §6 门槛），僵局率下降。

**非目标**
- 不改跳杀规则本身、不改胜负目标（仍 4 跳杀 / 4 感染）。
- 不引入联网/存档/音效/大模型。
- 不做与本目标无关的重构。

## 3. 规则 R1 — 僵尸互斥

### R1.1 定义
任一僵尸经由 **移动**、**召唤**、**感染生成** 落子后所占据的格，**不得与另一僵尸正交（上下左右 4 向）相邻**。
- "另一僵尸"：落子判定**当刻**棋盘上的其他僵尸，**不含正在移动的该僵尸自身**（其原格在判定时视为已空）。
- 2 步移动：**每一步**独立按当步棋盘状态判定（第 1 步落定后形成的新局面用于第 2 步判定；本回合已移动的僵尸按其新位置计）。
- 仅约束**僵尸–僵尸**正交相邻；僵尸与幸存者正交相邻不受限（否则感染无法成立）。

### R1.2 为什么不破坏感染（必须成立的不变量）
感染需 2 只僵尸正交贴住**同一**幸存者。幸存者的 4 个正交格两两之间的曼哈顿距离恒为 2（正对或斜对），**没有任何两个彼此正交相邻**。故任意合法感染阵型中的两只僵尸天然不正交相邻，R1 不阻挡感染。**实现完成后必须有测试断言此不变量（见 §5 S-3）。**

### R1.3 落地点（代码）
- `src/game/rules.ts::legalMoves`：当 `piece.side==='zombie'`，过滤掉"目标格正交相邻存在其他僵尸"的着手。须正确处理移动僵尸自身原格视为空。
- 召唤候选：`handleZombieClick` 召唤分支 + AI `enumerateZombie` 的 summon 候选，过滤同一谓词。`emptyCells` 是通用函数，**不在 `emptyCells` 内改**；新增一个僵尸感知的谓词 `zombieMayOccupy(state, r, c, ignoreId?)` 供 move/summon/spawn 共用。
- AI `enumerateZombie` 走 `legalMoves`/共用谓词，自动继承约束。

### R1.4 死锁兜底
若僵尸方在某回合**不存在任何合法行动**（无可召唤点、无任一僵尸可合法移动）：该僵尸回合**跳过**（`runEndOfTurn` 正常结算并切边），不得卡死或抛错。`main.ts::aiFallbackEndZombie` 同步遵守 R1，且在无合法手时安全结束回合。

## 4. 规则 R2 — 感染生成异地化

### R2.1 定义
幸存者被感染移除且 `zombieReserve > 0` 时，新僵尸**不在被感染格原地生成**，改为在一个**可用点**生成：
- 可用点判据：**与现有召唤"可放置格"完全同源**（即 `emptyCells` 的判据，不在此 spec 重新定义其对 stone/house/占用的处理，避免双份定义漂移）**且**满足 R1（不与任何僵尸正交相邻）。判定基于**本次感染结算后**的局面（被感染幸存者已移除、施感染的僵尸在位）。实现时若 `emptyCells` 口径需复用，抽成单一谓词供召唤与生成共用。
- 选择算法（**确定性**）：在所有可用点中，取"到最近**存活**幸存者（本次移除后仍在场者）曼哈顿距离最小"的点；多点并列时取 `(r, c)` 字典序最小者。
- 退化：无存活幸存者，或无任何可用点 → **跳过生成**（幸存者仍移除、`zombieKills` 仍 +1、`zombieReserve` 不减）。
- `zombieReserve` 仅在确实生成时 −1。

### R2.2 落地点
改在 `src/game/rules.ts::runInfection`。该函数被 `state.ts::runEndOfTurn` 与 AI `ai.ts::simulate` 共用 → AI 前瞻自动吃到新行为。**选择算法必须确定性**（`cloneState`/`simulate` 用于打分，非确定性会使 AI 评估抖动、`scripts/test_*` 不可复现）。

### R2.3 多重感染的顺序
同一回合末若多个幸存者同时满足感染（现规则"同回合只扫一次、不连锁"保持不变）：逐个结算，**每个**感染各自按 R2.1 选点（后一个的"存活幸存者"与"在场僵尸"集合反映前一个已结算的结果）。结算顺序按 `state.pieces` 中幸存者的稳定顺序（确定性）。

## 5. 纯函数 seam + 测试（testable enumeration）

新增 DOM 无关纯函数并配 `npx tsx` 契约测试（沿用 `scripts/test_*.ts` 既有惯例）：

- **S-1 `zombieMayOccupy(state, r, c, ignoreId?)`**（rules 层）：返回该格是否可被某僵尸占据（满足 R1）。测试：被另一僵尸正交相邻 → false；仅斜相邻 → true；`ignoreId` 指定的僵尸不计入；与幸存者正交相邻不影响结果。
- **S-2 `pickInfectionSpawn(state)`**（rules 层，给定已移除被感染者、僵尸在位的局面）：返回生成点或 `null`。测试:确定性(同输入同输出);取最近存活幸存者;tie-break (r,c);无可用点→null;无存活幸存者→null。
- **S-3 感染不变量测试**：构造 6 种"2 僵尸夹 1 幸存者"几何（N/S、E/W、N/E、N/W、S/E、S/W），断言两僵尸均满足 `zombieMayOccupy`（即 R1 不阻挡任何合法感染阵型）。这是 G-B 的回归守卫。
- **S-4 破墙断言**：构造"贴边僵尸列"在 R1 下只能成棋盘格，断言存在某幸存者位置使 `legalJumps` 返回对该列某僵尸的跳杀（贴边墙不再免疫）。这是 G-A 的回归守卫。
- 现有 `scripts/test_card_modal.ts`、`scripts/test_start_menu.ts` 必须仍全过；新增上述测试加入手测清单。

## 6. harness 验收门槛（量化，可测）

跑 `npx tsx scripts/simulate.ts`（每配置 ≥100 局，必要时调大）。先记录**改前基线**数字，再以下列门槛验收（数字硬门槛，非主观）：

- **G1 平衡**：`easy/easy` 幸存者胜率 ∈ [45%, 55%]；任一侧不 > 60%。
- **G2 反僵局（核心）**：`easy/easy` 与 `hard/hard` 的平局/僵局率**严格低于改前基线**。
- **G3 感染仍可达**：AI-vs-AI 全程仍能产出感染（zombieKills 分布非塌缩为 0；与基线同量级或受控）。
- **G4 破墙**：S-4 通过；附加场景脚本——贴边僵尸阵在新规则下被 AI 幸存者在有限回合内跳杀，不再 0:0 拖死。
- **G5 工程**：`npx tsc --noEmit` 干净；S-1..S-4 + 既有两套测试全过。

未达 G1/G2 时，**调 `ai.ts` 的 `zombieScore`/`scoreAction` 进攻权重补偿**（见 §7），权重由 harness 迭代调出，不预设。门槛全绿方可视为完成（遵循 verification-before-completion：报真实数字，不口头声明）。

## 7. AI 进攻 retune（与 R1/R2 绑定）

R1+R2 是多轴削僵尸（无法结实推进 + 感染雪球的"原地+1"消失），不配 AI 变敢打则平衡必崩；这同时解决原始诉求"AI 太胆小"。方向（具体权重为 harness 调参产物）：
- 新增僵尸进攻/合拢偏置，对称现有 `survivorOffense`，在 `scoreAction` 的 hard worst-case **之后**施加（仅对 zombie 侧），奖励：贴成 1-正交（将成感染）、压缩幸存者合法移动数（窒息）、向边角驱赶。
- 压低过强的恒定 `−killable.size*160`：当被暴露的同一僵尸正贡献夹击/逼近时，降低其惩罚权重（接受"舍 1 换感染"的正期望交换）。
- 不改 `easy` 的 `EPS` 噪声语义；medium 仍隐藏。

## 9. 风险 / 决策记录

- **R-1 多轴削僵尸**：R1+R2 叠加显著削弱僵尸。缓解：§7 retune + §6 硬门槛，**必须同 spec 同批实现**，不可只做规则不做 retune。
- **R-2 手感变化大**：R1 约束每一步僵尸移动（"僵尸互相排斥"），比仅改边缘的方案手感变化更大——已与用户对齐接受，作为破墙的统一解。
- **R-3 确定性要求**：R2 选点与多重感染顺序必须确定性，否则 AI `simulate` 评估抖动。S-2 测试守卫。
- **决策**：感染生成点选"靠最近存活幸存者"（用户定，含义：保压力以部分抵消 R1/R2 削弱）。
- **决策**：约束范围"全都管"（移动+召唤+感染生成；感染生成靠异地化天然满足）。
