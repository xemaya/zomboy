# ZOM-BOY+ Web 版 — 设计（v2，对齐 canonical 规则）

源参考：用户口述完整规则 + `ChatGPT Image May 15, 2026, 01_12_32 PM.png` 视觉。
v1 规则有偏差（卡牌 / 全僵尸上场 / 30 回合上限），v2 全部修正。

## 1. 范围
- 浏览器单页热座对战，2 人轮流
- 像素美术：DeerAPI gpt-image-2 生成 sprite
- 含**地图生成器**：解决原图地图过密的 bug
- 不做：AI、联网、存档、声音

## 2. 技术栈
- Vite + TypeScript（无框架，原生 DOM）
- CSS Grid 渲染棋盘
- 像素字体 `Press Start 2P` 仅用于标题/数字；正文用系统中文/英文 sans
- 端口 `1424`

## 3. 棋盘
- **8×8** 网格
- 4 个 `START` 在四角，开局 4 个幸存者就位
- 4 个房子，每象限 1 个
- **僵尸开局 0 个在场，库存 9**
- 石头按生成器约束放置

## 4. 地图生成器
- 锁定 4 START（角）
- 房子：每象限 2×2 候选区随机选 1 个
- 石头：剩余空格按 `stoneRatio = 0.18` 随机
- 校验：每个 START 用 BFS（4 邻接，跳过 stone）可达每个房子 + 每个其他 START
- 每个 START ≥2 个非石头邻居
- 开放格数 ≥ 9（库存）+ 4（幸存）+ 6（缓冲）
- 失败最多重抽 200 × 5 个递减 stoneRatio

## 5. 规则（canonical）

### 5.1 回合
- 僵尸先手；双方轮流
- 一回合（round）= 一方完成自己回合 → 对方完成 → +1

### 5.2 幸存者回合（每回合必行）
- **移动 1 个幸存者**
- 方向：上下左右 + 斜走（8 方向，1 格）
- 不能进入石头 / 其他棋子的格子
- **跳杀（核心）**：相邻格（8 方向）有 1 个僵尸 且 该方向再过去 1 格是合法空地 → 跳过去，被跳的僵尸死亡，survivor 得分 +1
- **房子触发**：幸存者落在房子格（任何方式：普通走 / 跳杀落点 / 骰子奖励走）→ 立即停下并掷 D6
  - MVP 取舍：房子**一次性**消耗（防 cheese）
  - 卡牌/骰子触发的奖励走**不再**踩房子（即奖励走不触发新的掷骰）

### 5.3 僵尸回合（二选一）
- **方案 1：召唤**（仅库存 > 0）
  - 放 1 个新僵尸到任意空草地，库存 -1
  - 库存 = 0 后此方案不可用（**取消了原文字规则的「传送到任意空地」，瞬移过强**），僵尸只能用方案 2
- **方案 2：移动 2 次**
  - 共 2 次行动点，可分给 1 个或 2 个僵尸
  - 僵尸只能上下左右，1 格 / 行动点
  - 不可斜走，不可踩石头 / 棋子

### 5.4 感染（每回合末扫描）
- 每个幸存者：上下左右（仅正交） ≥ 2 个僵尸 → 被感染
- 立即移除幸存者，zombie 得分 +1
- 库存 > 0 → 原地立刻生成新僵尸（库存 -1）
- 库存 = 0 → 格子变空
- 同回合内只扫一次（不连锁）

### 5.5 胜负
- **幸存者胜**：survivorKills = 4（跳杀满 4）
- **僵尸胜**：zombieKills = 4（感染满 4）
- 谁先达成谁立刻赢

### 5.6 事件卡（房子抽，非骰子；早期 spec 的 D6 已废）
开局洗 5 张：鬼魂 ×2、靴子 ×2、咖啡 ×1。幸存者踩房子抽顶牌，房子一次性。

| 名 | 效果 |
|----|------|
| 鬼魂 | 选一只僵尸**永久驱散**（从场上彻底移除，**不回库存**，非幸存者得分）。balance 调整：原「传送到空地」过弱，2026-05-15 改为永久移除 |
| 靴子 | 立刻让任一幸存者再走 1 格（正常走法，可跳杀） |
| 咖啡 | 立刻让任一幸存者连走 2 格 |

奖励走支持跳杀（可触发 +分），但不与房子交互（不再抽卡）。
鬼魂抽到时若场上无僵尸 → 卡牌失效（不卡死）。

## 6. UI 流程

### 6.1 僵尸回合
1. 顶部出现「召唤 / 移动 2 次」两个按钮
2. 选「召唤」：
   - 库存 > 0：高亮全部空地 → 点空地放置 → 回合结束
   - 库存 = 0：先选 1 个僵尸（高亮所有僵尸）→ 再选空地 → 回合结束
3. 选「移动 2 次」：进入选棋状态，行动点 = 2
   - 点僵尸 → 高亮合法移动 → 点目标格 → 行动点 -1
   - 重复直到行动点 = 0 → 回合结束

### 6.2 幸存者回合
- 点幸存者 → 高亮 (移动格 ∪ 跳杀落点)
- 点目标：自动判断是跳杀还是普通移动；杀僵尸记录得分
- 若落入房子 → 弹掷骰 modal，确定后选目标按骰子效果

### 6.3 HUD
- 上方 Turn Strip：回合数 / 当前阵营 / 当前阶段提示 / 库存 / 房子剩余
- 右上：得分板（survivorKills / zombieKills，各 / 4），场上人数 + 库存
- 右下：日志（最近 18 行）
- 卡片模态：骰子结果 + 效果 + 确定（确定后骰子模态变为 step toast 不挡棋盘）

## 7. 视觉规范
- 标题 `Press Start 2P` 28-32px，单层粉色阴影（无双层重影）
- 正文/列表 `Noto Sans SC` 13-14px
- 骰子点数 64px Unicode 点数符号
- 配色：暗背景 `#2a2724`，cream 面板 `#f6f1e0`，pink accent `#c93f74`，GB 绿 `#0f380f-#9bbc0f`

## 9. 验证（手测）
- 5 次重新生成地图皆可玩
- 僵尸：召唤、库存空后只能移动 2 次（1+1 / 2+0）都跑通
- 幸存者：正交、斜走、跳杀（含 8 方向）都跑通
- 骰子 1/2/3 都触发鬼魂，4/5 触发靴子，6 触发咖啡
- 跳到 4 杀僵尸 → 幸存者胜利 overlay
- 感染到 4 → 僵尸胜利 overlay
